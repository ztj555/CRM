# 融鑫CRM局域网版 - 项目规格文档

> 基于融鑫汇-客户管理系统（客帮手）深度分析复刻
> 目标：局域网部署，支持多账号，界面功能与原系统基本一致

---

## 一、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Vite + Element Plus | 现代前端框架，组件丰富 |
| 后端 | Spring Boot 3 + JPA | 稳定可靠，开发效率高 |
| 数据库 | MySQL 8 / H2（开发） | MySQL用于生产，H2用于快速启动 |
| 认证 | JWT | 无状态认证，支持多账号 |
| 文件存储 | 本地磁盘 | 附件上传到本地目录 |
| 部署 | 一台电脑运行jar + vue build | 局域网即可访问 |

## 二、数据库核心表结构

### 2.1 用户表（users）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| username | VARCHAR(20) UNIQUE | 账号（手机号） |
| password | VARCHAR(128) | bcrypt加密 |
| real_name | VARCHAR(50) | 真实姓名 |
| role | TINYINT | 1=顾问 2=主管 3=管理员 |
| dept_id | BIGINT | 部门ID |
| daily_quota | INT | 日分配上限 |
| accept_new_data | TINYINT | 1=接受新数据 0=关闭 |
| status | TINYINT | 1=正常 0=禁用 |
| created_at | DATETIME | 创建时间 |

### 2.2 部门表（departments）
| 字段 | 类型 |
|------|------|
| id | BIGINT PK |
| name | VARCHAR(100) |
| parent_id | BIGINT（0=顶级）|
| level | TINYINT（1公司/2大区/3部门）|

### 2.3 客户主表（customers）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| phone | VARCHAR(20) | 手机号 |
| phone_hash | CHAR(64) | 查重用哈希 |
| name | VARCHAR(50) | 姓名 |
| gender | TINYINT | 0未知/1男/2女 |
| city | VARCHAR(50) | 城市 |
| age | TINYINT | 年龄 |
| source | VARCHAR(100) | 数据来源 |
| loan_type | TINYINT | 贷款类型 |
| apply_amount | DECIMAL(10,2) | 申请额度（万）|
| status | TINYINT | 客户状态（见状态枚举）|
| star_level | TINYINT | 星级 0-6 |
| is_blacklisted | TINYINT | 是否黑名单 |
| is_locked | TINYINT | 是否锁定 |
| is_important | TINYINT | 是否重要 |
| last_remark_at | DATETIME | 最后备注时间 |
| created_at | DATETIME | 创建时间 |

**资质字段（JSON存储）**：
- 婚姻状况、学历、职业类型、工资发放方式
- 房产类型、房产省市、抵押状态、是否按揭
- 车产类型、车牌省市、使用年限、车辆价值、按揭状态
- 保单情况、保险公司、保单缴费次数、社保、公积金
- 微粒贷额度、征信等级、信用卡额度、近6月查询次数、当前贷款笔数
- 负债类型、负债金额
- 客户资质文本

### 2.4 分配表（customer_assignments）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| customer_id | BIGINT | 客户ID |
| advisor_id | BIGINT | 顾问ID |
| pool_type | TINYINT | 1=我的客户 3=公共池 4=必跟进 |
| assigned_at | DATETIME | 分配时间 |
| assigned_by | BIGINT | 分配操作人 |

### 2.5 备注表（customer_remarks）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| customer_id | BIGINT | 客户ID |
| advisor_id | BIGINT | 顾问ID |
| content | TEXT | 备注内容 |
| remark_type | TINYINT | 0=跟进 1=主管点评 2=备忘 |
| created_at | DATETIME | 创建时间 |

### 2.6 在审件表（loan_cases）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| customer_id | BIGINT | 客户ID |
| advisor_id | BIGINT | 顾问ID |
| dept_id | BIGINT | 部门ID |
| bank_name | VARCHAR(100) | 进件银行 |
| apply_amount | DECIMAL(10,2) | 进件额度 |
| fee_rate | DECIMAL(5,4) | 费率 |
| stage | TINYINT | 1审核/2批款/3收款/4拒批/5违约 |
| approve_amount | DECIMAL(10,2) | 批款额度 |
| approve_at | DATETIME | 批款时间 |
| collection_amount | DECIMAL(10,2) | 收款金额 |
| collection_at | DATETIME | 收款时间 |
| submit_at | DATETIME | 提交时间 |

## 七、部署方案

### 7.1 单机部署（局域网）
- 后端：Spring Boot jar包，Windows服务或直接java -jar运行
- 前端：Vue build产物，Nginx或直接由Spring Boot静态资源服务
- 端口：后端8080，前端80
- 数据库：MySQL 8（与后端同机或局域网另一台）

### 7.2 访问方式
- 局域网IP访问：http://192.168.x.x（需查本机IP）
- 同事电脑浏览器直接访问

**第二期（后续迭代）**：
