## 一、系统概览

### 1.1 系统定位
融鑫汇CRM系统是一款面向**金融贷款行业**（信用贷、抵押贷等）的顾问团队客户管理系统，主要服务于：
- 贷款顾问（一线销售人员）的日常客户跟进
- 主管/团队长对下属团队的管理与监控
- 业务数据统计与绩效分析

### 1.2 整体架构
系统采用经典的 **左侧导航栏 + 顶部操作栏 + 内容区（iframe嵌套）** 结构，基于 Web 技术开发。

```
┌─────────────────────────────────────────────────┐
│  [Logo: RXH 融鑫汇]                              │
├──────────────┬──────────────────────────────────┤
│  用户信息卡  │   顶部操作栏                      │
│  (头像/姓名) │   [绩效目标] [公告] [设置]         │
├──────────────┤                                  │
│  左侧导航菜单│   内容区（iframe）                 │
│  - 我的客户  │                                  │
│  - 再分配客户│                                  │
│  - 公共池客户│                                  │
│  - ...       │                                  │
└──────────────┴──────────────────────────────────┘
```

### 1.3 主要角色（完整5级体系）

| 角色 | 层级 | 说明 | 不受限制项 |
|------|------|------|-----------|
| **负责人/总账号** | 最高 | 拥有系统全部权限，可切换到任意顾问账号 | IP白名单、夜间登录、客户上限、所有开关 |
| **区长（大区主管）** | 一级 | 管理整个大区，跨部门权限 | 按设置 |
| **部长/二级主管** | 二级 | 管理团队 | 按设置 |
| **主管** | 普通主管 | 管理顾问 | 按设置 |
| **顾问（不含学徒）** | 员工级 | 一线销售，有独立客户池 | 无 |
| **学徒/入司** | 实习级 | 不享受顾问权限，受更多限制 | 无 |

> 注：负责人（总账号）权限最高，可以切换到任意顾问账号进行操作，不受任何开关限制。

展示被**从其他顾问转分配**到当前顾问名下的客户。这类客户通常是被其他顾问跟进后未成单，再次分配给新顾问的数据。

### 3.3 公共池客户

**入口**：`/manage/kefu_clientsggc/index.html`  
**标题**：客帮手-公共池客户  

#### 3.3.1 功能概述
"公共池"是系统的**客户资源池**，所有未被分配或从顾问池中退出的客户都存放于此。当前公共池共有约 **148,535** 条客户数据。顾问可以从公共池中认领客户。

#### 3.3.2 特有筛选条件

| 筛选项 | 说明 |
|--------|------|

```
拨打电话 → 记录备注（更新状态：未接通/待跟进/有意向等）
    ↓
多次跟进后 → 邀约上门（状态：预约上门）
    ↓
上门面谈 → 评估贷款资质（星级调整：0-6星）
    ↓
资质符合 → 进件（添加在审件）
    ↓
银行审核（进度：审核→批款→收款）
    ↓
放款成功 → 收款 → 创收确认
```

### 6.2 数据流转流程

```
原始数据（Excel导入/API）
    ↓
公共池（约15万条待分配）
    ↓
系统/主管分配
    ↓
顾问"我的客户"（上限400条）
    ↓
[长时间未跟进] → 抓回公共池
[离职顾问数据] → 离职人数据分配
[主管转移] → 再分配客户
```

### 6.3 星级评定标准

| 星级 | 含义 |
|------|------|
| 0星 | 未评级/无意向 |
| 1星 | 轻微意向 |
| 2星 | 有一定意向 |
| 3星 | 可贷（3星以上为优质客户）|
| 4星 | 高意向 |
| 5星 | 非常高意向 |
| 6星 | 极高意向（特殊标记）|

## 十一、后续开发参考建议

基于对本系统的深度分析，如需开发高度类似的CRM系统，建议重点关注以下核心设计：

1. **客户状态机**：设计清晰的客户状态流转（待跟进→有意向→预约上门→已上门→进件→批款→收款）

7. **数据来源追踪**：每条客户数据保留来源标签
8. **通话系统集成**：与电话系统对接，自动记录通话数据
9. **iframe式导航**：多模块共享左侧导航，内容区动态加载

- 按状态（待跟进/待签约/已放款/已上门/已签约各设天数阈值）

**谁名下数据会被抓**（`intoggcpeople`）：3种选择

**不抓数量**：`notoggc_num`（number）锁定客户数

### 16.9 公共池可见性

|--------|------|
